Technical Blog: SQL Query Optimization

April 19, 2015

When working with big database you would want retrieve data efficiently and fast. Here are some SQL command techniques that might help you.

It is faster if you type the actual columns names instead of '*' in SELECT statement. For example, instead of: 


                SELECT * FROM students;

try 


                SELECT id, first_name, last_name, age, subject FROM students;

Below are some examples with WHERE clause. Instead of: 


                SELECT id, first_name, age FROM students WHERE age != 10;

try 


                SELECT id, first_name, age FROM students WHERE age > 10;

Instead of: 


                SELECT id, first_name, age FROM students 
                WHERE SUBSTR(first_name, 1, 3) = 'Jo';

try 


                SELECT id, first_name, age FROM students 
                WHERE first_name LIKE 'Jo%';

Instead of: 


                SELECT id, first_name, age FROM students 
                WHERE first_name = NVL (:name, first_name);

try 


                SELECT id, first_name, age FROM students 
                WHERE first_name LIKE NVL (:name, '%');

Instead of: 


                SELECT product_id, product_name FROM products
                WHERE unit_price >= MAX(unit_price) and unit_price <= MIN(unit_price);

try 


                SELECT product_id, product_name FROM products
                WHERE unit_price BETWEEN MAX(unit_price) and MIN(unit_price);

Instead of: 


                SELECT id, name, salary FROM employee 
                WHERE dept || location= 'ElectronicsBrooklyn';

try 


                SELECT id, name, salary FROM employee 
                WHERE dept = 'Electronics' 
                AND location = 'Brooklyn';
