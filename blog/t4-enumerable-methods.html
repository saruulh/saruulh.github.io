Technical Blog: Enumerable Method #map

March 22, 2015

Enumerables are modules, which contain methods that are packaged together to work with other classes as long as the class that's using the enumerable has its own #each method. In other words, #each method works as a connector between the module and class so that class can access to those module methods.

Enumerable enables us to go through every element in a collection object, like an array or a hash.

In this case we can use #map method (alias of #collect) to go through each element. Let's look at an array for an example:

["blue", "red", "purple", "yellow"].map { |x| p "#{x} color"}
=>
"blue color"
"red color"
"purple color"
"yellow color"
=> ["blue color", "red color", "purple color", "yellow color"]
 
["blue", "red", "purple", "yellow"].map { |x| p "#{x} color.".capitalize }
=>
"Blue color."
"Red color."
"Purple color."
"Yellow color."
=> ["Blue color.", "Red color.", "Purple color.", "Yellow color."]

As you can see, #map always returns an array (a new object) with an accumulated result of a given code block in the same size as the original enumerable.

But what if we used #puts in the code block in the above example, like so:

["blue", "red", "purple", "yellow"].map { |x| puts "#{x} color"}
=>
blue color
red color
purple color
yellow color
=> [nil, nil, nil, nil]
 
["blue", "red", "purple", "yellow"].map { |x| puts "#{x} color.".capitalize }
=>
Blue color.
Red color.
Purple color.
Yellow color.
=> [nil, nil, nil, nil]

The #puts method returns nil in this case, because the return value of puts is always nil, so keep that in mind!
