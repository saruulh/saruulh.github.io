Technical Blog: Ruby Classes

March 29, 2015

Classes are collection of variables and methods that behave certain way. It’s a way to collect and hold bunch of different methods and variables in one place and essentially produce a result under a common goal.

You can declare a class by class keyword with any name you choose as long as it starts with capital letter or camel case (CamelCase) for two word names. Classes ends with end keyword.

class CalculateSqft
end

Declaring methods within the class are the same way we create regular methods by defining a method def keyword, name and an end keyword. Methods that are inside a class is an instance methods and can be accessed or shared by any other instances of the class.

class CalculateSqft
 def area
  width * height
 end
end

But above method called area that calculates width and height needs to store that calculation value somewhere. That’s when instance variable comes in. Instance variables always starts with @ sign and it can be shared with other methods within that same class.

class CalculateSqft
 def area
  @sqft = width * height
 end
end

If you so desire, you can have a special method called initialize that executes every time a new class instance is created. For example, let’s print a message every time a new CalculateSqft object is created.

class CalculateSqft
 def initialize
  puts “Hey, what’s your width and height?”
 end
 def area
  @sqft = width * height
 end
end
