Technical Blog: Ruby Doc vs JavaScript Doc

April 12, 2015

Both the Ruby and JavaScript doc are very dense and difficult to understand for a first time learner. However, as you go along and find things, you start to discover some solid ground.

Ruby Doc

The Ruby Doc was not like reading a book or magazine, it was a highly technical and dense document that was very intimidating. One of the hardest parts was getting over this. I would assume it's much like reading Shakespeare. The language is tough as first, but after a while, you start to figure out what world you are in.

What I found helpful was to make sure what version you're working with. When you are working with the latest version of Ruby, some of methods in the earlier versions of Ruby don't apply. When you search what you're looking for, you should start with the summary of its description followed by some examples. This helped me get my eyes used to the syntax examples that are given.

I would highly recommend reading Maria Piper's blog post on how to read a Ruby Document and utilize it efficiently. You can find her post [*here*](http://mariabpiper.tumblr.com/post/116045895341/ruby-docs-for-newbs).

Another thing I found helpful is to access Ruby Documentation descriptions with ri command from your terminal (this is also helpful if you find yourself with no internet connection).

Javascript Doc

The Javascript document was a similar experience as far as the density and difficulty in reading the information. It's not, by any means, a casual read. One must be an active reading to fully understand the information.

However, I found the examples are easier to understand. Their detailed examples are more descriptive. This allows the reader to actually see what certain methods can do for you.

Since I just was introduced to Javascript this week, I don't have the experience that I have with Ruby, so I can't get deeper into specifics. However, with the greater explanations, I think this is going to be a bit easier on my mind.
