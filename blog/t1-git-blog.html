As a beginner, joining and going through Dev Bootcamp's first week was, well, like a "bootcamp". I learned so much this week than I ever did. I got to learn the whole ecosystem of development and how each component is a vital part of the whole. I had a very limited idea of Git and Github (yes they are two different things) but now I know Git is a software that you can install on your computer locally which controls and manages your file versions (version control system). Simply put, it saves and keeps snapshots of your file changes in your computer. Github is a remote centralized hub where all files saved, organized and merged without conflicting, deleting or overwriting your files. Sound neat? So, this is an important place to collaborate and work on a same project with project team members. You can do whole lot of things here and it seems pretty deep, but this is a good place to start. The benefits of version control are tremendous, let's say you spent hours writing code and you accidentally deleted few lines, you can revert back to your earlier state with version control and recover.

Git keeps an eye on files you are changing and let's you know what files are changed by explicit commands, e.g. git status will show you which file is changed. It also shows you instructions what to do next. Your next thing would be to stage your changed files to be commited. You will be able to stage your files by git add file_name. Once added om the stage, you will be able to fully commit your files. Committing is like a saving or recording point for your file changes. Once you committd, now it ready to be pushed to the hub. You do this by typing git push origin master.

GitHub on the other hand, is a hub where you upload your committed file (like we did by git push...) so that everyone else can view, edit, share and have a discussion on the file that you saved.
