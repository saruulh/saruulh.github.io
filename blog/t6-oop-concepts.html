Technical Blog: Variable Scope

April 5, 2015

Ruby has scopes that define variable accessibility. Ruby has global, class, instance, local and constant variables. Each variables has their own special naming conventions. For example:

$global variable - starts with $ sign with lower case letters @@class variable - starts with double @ sign with lower case letters @instance variable - starts with single @ sign with lower case letters local variable - starts with lower case letters or _ CONSTANT variable - with ALL CAPS

Global variables are accessible from anywhere in the program. It's used rarely and discouraged. Because it is accessible from anywhere in the program and you can alter its value anywhere in the program, thus it would affect other objects behave unexpectedly. For example:

$hello = "Hello, world."

Class variables are accessible within a class and its all instances. Similar to global variable but within its class. That said, the same goes for its value alteration. If the value is modified it will affect all other objects using this class variable. Class variables must be initialized at the creation. For example:

@@hello = "Hello, world."

Instance variables are widely used. It's accessible within its class it belongs to and is similar to class variables, but its values are local and specific to that instance. For example:

@hello = "Hello, world."

Local variables that are declared in a method or a loop and are accessible only within that method or a loop. You cannot access it outside of that loop or method. For example:

hello = "Hello, world."

Constants store values and once assigned a value, it should not be altered. Its values are alterable, but Ruby will throw a warning message. Similar to global variable and are accessible within the context of that class or module.

HELLO = "Hello, world."
